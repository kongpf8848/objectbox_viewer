## 简介
本文件面向 ObjectBox Viewer 的开发者与调试人员，系统化梳理仓库中的开发与调试工具链，覆盖数据解析调试、诊断工具、页面与模式识别、错误排查与性能分析等主题。文档以"工具—用法—输出—最佳实践"的方式组织，帮助读者快速定位问题、验证假设并优化数据库读取与展示流程。

**更新** 本版本大幅扩展了调试工具集，新增了多个专门的调试工具文件，为数据库分析和解析提供全面的调试能力，包括模式详情分析、当前状态检查、全量数据扫描等功能。

## 项目结构
该项目采用 Flutter 应用结构，配合一组独立的 Dart 调试脚本（位于 tool/ 目录），用于直接解析 ObjectBox/LMDB 数据文件，辅助 UI 层的数据浏览与诊断。

- 应用层
  - 入口与界面：lib/main.dart、lib/widgets/home_page.dart
  - 状态管理：lib/bloc/db_bloc.dart
  - 模型定义：lib/models/objectbox_model.dart
  - 数据服务：lib/services/objectbox_service.dart
- 工具层（调试脚本）
  - 页面与页眉解析：tool/debug_pages.dart、tool/dump_pages.dart、tool/parse_pages.dart、tool/trace_pages.dart、tool/find_page.dart、tool/debug_all_pages.dart
  - 数据转储与 FlatBuffer 解析：tool/debug_dump.dart、tool/debug_diagnostic.dart、tool/hex_dump.dart、tool/dump_entries.dart、tool/debug_all_data.dart、tool/debug_values.dart
  - 模式与架构分析：tool/debug_schema_detail.dart、tool/debug_current.dart、tool/debug_schema.dart、tool/debug_schema_parse.dart、tool/debug_schema_raw.dart
  - 特殊场景调试：tool/debug_branch.dart、tool/debug_deleted.dart、tool/debug_meta.dart、tool/debug_keys.dart、tool/debug_subdb.dart、tool/debug_standard.dart
  - 集成测试与服务调用：tool/debug_parse.dart

## 核心组件
- 应用入口与导航
  - 提供打开数据库目录、状态栏提示与底部状态条，支持通过文件选择器定位 data.mdb 所在目录，并自动探测子目录中的数据库文件。
- 状态管理（BLoC）
  - 统一处理打开数据库、选择实体、刷新数据与关闭数据库等事件，维护加载、成功、错误三种状态。
- 模型定义
  - 定义实体、属性、索引、关系等模型，支持从 JSON 或直接从数据库中发现模型。
- 数据服务
  - 直接解析 data.mdb 文件，无需 objectbox-model.json；支持发现模型、读取实体数据、去重保留最新写入版本。
- 调试工具
  - 页面扫描、页头打印、页大小猜测、FlatBuffer 解析、字符串搜索、十六进制转储、条目导出等，覆盖从底层页到上层数据的全链路诊断。
  - **新增** 专门的模式分析工具（debug_schema_detail.dart）用于深入解析 FlatBuffer 架构细节
  - **新增** 当前状态检查工具（debug_current.dart）用于快速获取数据库当前状态概览
  - **新增** 全量数据扫描工具（debug_all_data.dart）用于全面分析数据库中的所有数据条目

## 故障排查指南
- 无法打开数据库
  - 确认选择了包含 data.mdb 的目录；若目录下无该文件，UI 会尝试在子目录中查找。

- 页大小不匹配
  - 使用页大小扫描与魔数校验，确认候选页大小集合中存在有效值。

- FlatBuffer 结构异常
  - 使用 FlatBuffer 解析脚本定位表头、VTable 与字段偏移，核对字段类型与长度边界。
  - 参考：[tool/debug_dump.dart:48-135](file://tool/debug_dump.dart#L48-L135)、[tool/debug_diagnostic.dart:257-329](file://tool/debug_diagnostic.dart#L257-L329)
- 实体名称或属性缺失
  - 若未提供 objectbox-model.json，服务层会通过字符串搜索与 VTable 扫描进行发现；必要时手动比对十六进制转储。

- 时间戳误判
  - 解析脚本会对大整数进行时间戳范围判断；若出现异常时间，请检查数据来源与单位。

## 结论
本仓库提供了从 UI 到底层解析的完整工具链：UI 层负责交互与展示，服务层负责解析与去重，工具层提供多维度的诊断能力。**更新后的版本大幅扩展了调试工具集，新增了多个专门的调试工具文件，包括模式详情分析、当前状态检查、全量数据扫描等，为数据库分析和解析提供了更全面的调试能力。**通过组合使用页面扫描、FlatBuffer 解析、模式分析与集成测试脚本，开发者可以高效完成数据解析调试、模式识别、架构验证与性能优化，从而提升整体开发效率与稳定性。

## 附录
- 参数与输出约定（通用）
  - 输入：data.mdb 文件路径（或目录路径，由 UI 自动探测）
  - 输出：结构化文本（页头、条目、实体/属性、字段类型与值、错误信息等）
- **新增** 专用工具参数约定
  - 模式分析工具：支持指定数据库路径，输出实体 ID、名称、属性列表等详细信息
  - 数据扫描工具：支持全量扫描和条件筛选，输出条目统计、重复检测等结果
  - 键值分析工具：输出十六进制键值、对象 ID 提取、实体 ID 等信息
- 最佳实践
  - 先用页面扫描与页大小检测确认基础结构
  - 再用 FlatBuffer 解析脚本验证实体与属性布局
  - 使用模式分析工具深入检查架构细节
  - 通过全量数据扫描验证数据完整性和一致性
  - 使用专用工具处理特殊场景（分支页、删除数据、元数据等）
  - 对于大文件，优先分页读取与限制输出数量，避免内存峰值
  - **新增** 结合多种工具进行交叉验证，确保分析结果的准确性
